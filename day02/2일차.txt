JSP기술이 나온 이유
 ( JSP 주된 목적은 웹브라우저에 보여줄  HTML 문서를 생성하는 것,  HTML을 생성하는것이 주된 목적 )


  서블릿이  html을 응답하는게 불편하기 때문에 나온 기술이다 
  HTML 안에 자바 코드를 끼워 넣을 수 있게 해서, HTML 작성은 그대로, 필요한 부분만 자바 코드로 처리할 수 있음.
  즉,  서블릿의 불편함(HTML 출력 코드 작성) 을 해결하려고 나온 기술이 JSP이다 
################################
JSP 동작원리

 
   jsp동작원리  :   
  JSP 페이지는 JSP 컨테이너(Tomcat 내부 JSP 엔진, Jasper 등) 에 의해  
  ->   Servlet 자바 코드(.java) 로 변환됨
  ->  그 후 .class 파일로 컴파일 → 서블릿 컨테이너가 이 서블릿 클래스를 로드해서 실행
  따라서 JSP는 “서블릿으로 변환된 후 실행되는 기술” 


 #####
요청흐름
######

1)  클라이언트 요청 → 브라우저가 hello.jsp 요청
2)  JSP 엔진(Jasper 등) 이 JSP 파일을 Servlet 자바 코드(.java) 로 변환
    예: hello_jsp.java
3) 변환된 .java 파일을 컴파일해서 .class 파일 생성
    예: hello_jsp.class
4)  서블릿 컨테이너(Tomcat)가 .class를 메모리에 로드해서 실행 
    실행 결과(HTML)를 브라우저에 응답
